# zerollama-dashboard

Одностраничная панель мониторинга для серверов [llama.cpp](https://github.com/ggml-org/llama.cpp).
Она позволяет просматривать состояние каждого слота запросов и параметры сэмплирования в режиме реального времени, а также получать подсказки на простом языке, когда что-то требует настройки.
Встроенная панель чата позволяет отправлять запросы и наблюдать за реакцией метрик прямо на той же странице.

[**▶ Попробовать интерактивное демо**](https://jungrok5.github.io/zerollama-dashboard/?demo=1) — работает на встроенном mock-сервере, реальный бэкенд не требуется.

## Зачем это нужно?

Встроенный веб-интерфейс llama.cpp представляет собой простой чат-клиент и не показывает текущее состояние сервера. Связка Prometheus + Grafana отображает цифры, но не подскажет, какие параметры запуска нужно изменить. Open WebUI, LibreChat и подобные инструменты — это клиентские интерфейсы для чата, а не мониторы состояния слотов. zerollama-dashboard восполняет этот пробел: это специализированный интерфейс оператора для `/metrics` + `/slots` с подсказками по оптимизации, поддержкой многомодельного режима **роутера** (включая отображение фактических аргументов запуска каждой модели) и панелью чата для тестирования сервера — и всё это в одном HTML-файле, который можно положить рядом с `monitor.html` или открыть напрямую через GitHub Pages.

## Что вы получаете:

- **Мониторинг в реальном времени** — скорость генерации, очередь запросов и активность слотов обновляются ежесекундно.
- **Встроенный чат** — общайтесь с моделью прямо на странице мониторинга. Поддерживаются стриминг ответов, рендеринг Markdown, кнопка остановки генерации и слайдеры для настройки `temperature`, `top_p` и `max_tokens`.
- **Подсказки на простом языке** — при высокой нагрузке на сервер панель подскажет, какой параметр оптимизировать (например: «очередь растет — попробуйте увеличить `--parallel` на 1»).
- **Детализация по слотам** — полная конфигурация сэмплирования (temperature, top_k, top_p, repeat_penalty, mirostat и др.) собрана в одном месте.
- **Два языка интерфейса** — Русский / English. Выбирается автоматически на основе настроек браузера, также язык можно переключить вручную в шапке страницы.
- **Автоопределение** режима работы сервера: одиночная модель или многомодельный роутер.

## С чего начать?

Вам понадобится запущенный сервер `llama-server` с флагом `--metrics`.
Самый простой вариант — настроить `llama-server` так, чтобы он сам раздавал файл `monitor.html` (или его улучшенную версию `monitor-enhanced.html`):

```bash
mkdir -p public
cp monitor-enhanced.html public/
llama-server -m model.gguf --metrics --port 8080 --path ./public
```

После этого откройте страницу `http://localhost:8080/monitor-enhanced.html`.

Для мониторинга сервера на другой машине разместите файл `monitor-enhanced.html` на любом статическом хостинге и укажите адрес сервера через параметр `?server=`:

```
http://localhost:8000/monitor-enhanced.html?server=http://10.0.0.5:8080
```

## Параметры URL

| Параметр | Что он делает |
|---|---|
| `server` | URL-адрес llama-server (по умолчанию: текущий источник) |
| `model` | Для режима роутера: какую модель выбрать по умолчанию |
| `lang` | Язык интерфейса (`en` / `ru`) |
| `demo` | `1` или `router` для запуска встроенного mock-сервера |

Все настройки хранятся прямо в URL-адресе, поэтому ссылка полностью фиксирует текущее представление — поделитесь ею, и получатель увидит то же самое.

## Опционально: лог сервера

Панель мониторинга может автоматически читать лог `server.log` при условии, что он находится на том же источнике (origin), что и HTML-файл. Для этого перенаправьте вывод llama-server в файл внутри обслуживаемой директории:

```bash
llama-server -m model.gguf --metrics --port 8080 --path ./public \
  > ./public/server.log 2>&1
```

Если лог-файл недоступен, панель чтения логов автоматически скроется — дополнительная настройка не требуется. Передайте `?log=path/to/file` для принудительного переопределения пути к логу или `?log=` для отключения панели.

## Конфиденциальность и безопасность

- Панель отправляет ваши данные только на указанный вами URL-адрес `llama-server` (и, опционально, запрашивает путь к файлу логов на том же источнике).
- Панель не использует cookies, localStorage или любые другие средства отслеживания.
- Любое действие, изменяющее состояние сервера (загрузка/разгрузка модели, сохранение/восстановление/очистка KV-кэша слота), всегда требует вашего подтверждения.
- Чат безопасно рендерит Markdown: сгенерированный моделью текст не сможет выполниться как сторонний HTML или скрипт.
